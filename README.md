# 氛围音乐 & 底栏美化 插件

## 安装
1. 把整个 mood-music-bar 文件夹放到酒馆的
   SillyTavern/public/scripts/extensions/third-party/ 目录下（没有 third-party 就自己新建）。
2. 重启酒馆，或在 扩展面板 → 右上角刷新按钮 重新加载扩展列表。
3. 打开 用户设置 → "Custom CSS" 区块，插件设置面板会插在它正下方，
   默认只显示标题一行，点一下标题展开/收起。

## 本轮修复的问题
1. 头像抓取路径：不再靠猜接口地址，改成直接从聊天记录里"已经渲染出来的头像"复制
   src（取最近一条角色消息 / 用户消息里 .avatar img 的真实地址），这是酒馆自己算出来的
   路径，一定是对的。找不到消息时才依次回退到角色列表缩略图、/thumbnail?type=avatar&file=。
2. 底栏定位：#custom-music-bar 用 CSS order: -9999 固定排在
   #nonQRFormItems 这个容器内的最前面，这是容器级别的定位规则，和发送按钮
   本身的位置/顺序完全无关——就算以后调整按钮布局，音乐条依然稳定停在最左边。
3. 暂停按钮：移到设置面板里（"暂停/播放"按钮），不再放在底栏。
4. 设置面板折叠：默认折叠，只显示"氛围音乐 & 底栏美化"标题，点击标题展开/收起，
   状态会保存。
5. 绑定美化后自动接管：只要绑定了 >=1 个美化，手动开关会被强制关闭并禁用（变灰），
   完全交给自动切换逻辑；绑定列表上方加了一个搜索框，方便美化多的时候快速定位。
6. 自动音源接口：换成经过实测能正常返回搜索结果的公开 Meting API
   （https://api.injahow.cn/meting/?server=netease&type=search&id=）。
   它返回的 url 字段本身是该接口的一个跳转地址，不是最终 mp3 直链，但可以直接当
   <audio src> 用——浏览器请求音频时会自动跟随跳转，实测可以正常播放。
7. 备用链接失败：给 <audio> 加了 error 监听。不管是自动搜索出来的曲目还是
   备用链接播放报错（链接失效、格式不支持等），都会自动尝试候选列表里的下一首；
   自动搜索的候选和备用链接都试过一遍还是不行，才会弹一次提示，不会静默卡死。
8. "未播放"占位符：去掉了，没有曲目信息时底栏的文字区域直接留空、不占位。

## 功能总览
- 绑定美化自动开关（面板里勾选想绑定的 UI Theme，多选，带搜索框）
- 底栏音乐条：角色头像 + 用户头像（小尺寸、轻微重叠），点角色头像=上一首，
  点用户头像=下一首，不额外画箭头
- 切换角色自动读取角色设定文本，用当前连接的模型总结氛围关键词去搜歌并自动播放
- 播完自动下一首，一轮播完重新搜索
- 设置面板里最多 5 条备用音源 URL + 暂停/播放按钮
- #nonQRFormItems 原有文字节点是"隐藏"（visibility+收起空间）而不是删除，
  插件关闭会原样还原

## 仍然需要你根据实际情况关注的点
- 自动播放限制：浏览器通常要求"先有一次用户交互"才允许自动播放音频，
  如果切换角色后没有声音，去设置面板点一下"暂停/播放"按钮即可正常播放之后的音乐。
- generateQuietPrompt 依赖你当前连接的对话后端，没连接任何 API 时会自动退化为
  用角色名做搜索词，不会报错卡住。
- .mesIDDisplay / .mes_timer / .tokenCounterDisplay 是 SillyTavern 消息模板里
  楼层号/生成耗时/token数对应的类名，如果你的版本改了类名，在 stripKnownSuffixes()
  里换成对应类名即可。
- 公开音源接口毕竟是第三方免费服务，不保证长期 100% 可用，这也是为什么专门做了
  备用音源功能兜底；如果长期不稳定，建议自行部署一份 Meting-API
  （GitHub: metowolf/Meting-API）替换设置面板里的接口地址。

## 关于"不影响其他美化"
所有视觉效果（底栏样式、隐藏文字、隐藏 s/t/#）都写在 body.mmb-active 作用域下，
JS 也只在 isPluginActive === true 时才会真正改写 DOM 内容；切换到未绑定的美化时，
会先移除 mmb-active 类、删除自己注入的 #custom-music-bar、把隐藏的文字节点还原，
不会残留任何影响。
