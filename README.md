# 氛围音乐 & 底栏美化 插件

## 安装
1. 把整个 `mood-music-bar` 文件夹放到酒馆的
   `SillyTavern/public/scripts/extensions/third-party/` 目录下（没有 `third-party` 就自己新建）。
2. 重启酒馆，或在 扩展面板 → 右上角刷新按钮 重新加载扩展列表。
3. 打开 用户设置 →（左侧一列）"UI Theme" 下方的 "Custom CSS" 区块，
   插件的设置面板会自动插入在 "Custom CSS" 输入框的**正下方**。

## 已实现的功能
- **绑定美化自动开关**：在设置面板里勾选你想绑定的 UI Theme（美化预设）。
  只要下拉框 `#themes` 切到被勾选的美化，插件自动启用；切到别的美化自动停用。
  如果一个都不勾，就退化为"自由开关"（面板顶部的手动开关）。
- **底栏音乐条**：通过 `$('#nonQRFormItems').prepend(...)` 固定在输入框最左侧，
  包含：角色头像 + 用户头像（小尺寸、轻微重叠）、暂停/播放按钮、曲名/歌手。
  - 点击**角色头像** = 上一首
  - 点击**用户头像** = 下一首
  - 不额外画左右箭头。
- **切换角色自动搜歌**：监听 `CHAT_CHANGED` 事件，读取角色 description / personality /
  scenario / first_mes，尝试用当前连接的模型（`generateQuietPrompt`）总结出 2~4 个
  氛围关键词，再拿去搜索音乐接口，自动播放，标题/歌手写进底栏。
- **播完自动下一首**；一轮搜索结果播完后会重新按角色氛围搜下一批。
- **备用音源**：设置面板里最多可添加 5 条你自己上传/获取的音频直链 URL，
  自动搜索失败时会自动切到备用列表播放，避免"找不到歌"卡住。
- **暂停按钮**：底栏内置，不影响原本发送框的输入/发送功能。
- **隐藏（不删除）`#nonQRFormItems` 原有文字节点**：插件激活时用 CSS
  `visibility:hidden` 隐藏，关闭插件会原样还原文字节点，不会丢内容。
- **隐藏计时器"s" / token计数"t" / 楼层号"#"**：通过 MutationObserver 直接
  改写 `.mes_timer` / `.tokenCounterDisplay` / `.mesIDDisplay` 的文本内容，
  只在插件处于激活状态时生效，切走绑定的美化立刻恢复原样。

## 你需要根据你的酒馆版本自行确认/调整的地方（无法 100% 保证，请测试后微调）
1. **音乐搜索接口**：默认填的是一个第三方公开接口示例地址，
   这类接口经常失效或有跨域限制，请自行更换成你确认可用的音源 API
   （设置面板里 "自动音源接口" 一栏可直接改）。这也是为什么专门做了
   "备用音源 URL" 功能，作为它挂掉时的保险。
2. **用户头像路径**：`index.js` 里 `updateAvatars()` 假设的路径是
   `/thumbnail?type=persona&file=...`，不同酒馆版本 persona 头像的接口路径可能不同，
   如果头像显示不出来，把这一行换成你版本里实际能取到用户头像 URL 的方式即可。
3. **`.mesIDDisplay` / `.mes_timer` / `.tokenCounterDisplay` 这三个类名**是 SillyTavern
   聊天消息模板里用来显示楼层号/生成耗时/token数的元素，如果你的版本改了类名，
   在 `stripKnownSuffixes()` 里换成对应类名即可。
4. **自动播放限制**：浏览器通常要求"先有一次用户交互"才允许自动播放音频，
   如果切换角色后没有声音，点一下底栏的暂停/播放按钮即可正常播放之后的音乐。
5. `generateQuietPrompt` 依赖你当前连接的对话后端，如果没连接任何 API，
   会自动退化为直接用角色名做搜索词，不会报错卡住。

## 关于"不影响其他美化"
所有视觉效果（底栏样式、隐藏文字、隐藏 s/t/#）都写在 `body.mmb-active` 作用域下，
JS 也只在 `isPluginActive === true` 时才会真正改写 DOM 内容；切换到未绑定的美化时，
会先移除 `mmb-active` 类、删除自己注入的 `#custom-music-bar`、把隐藏的文字节点还原，
不会残留任何影响。
